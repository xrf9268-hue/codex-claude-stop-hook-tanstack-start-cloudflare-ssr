# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
pnpm dev              # Vite dev server on :3000
pnpm build            # Vite build + tsc --noEmit
pnpm deploy           # Build + wrangler deploy to Cloudflare
pnpm check            # lint + typecheck + test:ci (full gate)
pnpm lint             # ESLint (flat config, v9)
pnpm typecheck        # tsc --noEmit
pnpm test:ci          # Vitest run (Cloudflare Workers pool)
pnpm test:hook        # Node native tests for hook logic
pnpm cf-typegen       # Regenerate worker-configuration.d.ts from wrangler.jsonc
```

Single test file: `pnpm vitest run test/worker.spec.ts`

## Architecture

**Runtime:** Cloudflare Workers with TanStack Start SSR.

**Entry flow:** `src/server.ts` exports `handler.fetch` from `@tanstack/react-start/server-entry` as the Worker fetch handler. Vite bundles via three plugins in order: cloudflare (SSR env) → tanstackStart → viteReact.

**Routing:** File-based via TanStack Router. Routes in `src/routes/` auto-generate `src/routeTree.gen.ts` (never edit). Root layout uses `shellComponent` in `__root.tsx` for SSR streaming.

**SSR data pattern:**
1. `createServerFn()` in `src/lib/queries/` defines server-side handler
2. `queryOptions()` wraps it for TanStack Query
3. Route loader calls `context.queryClient.ensureQueryData()` to prefetch
4. Component renders with `useSuspenseQuery()`

**API routes:** `src/routes/api/*.ts` use `createFileRoute` with `server.handlers.GET/POST`.

**Cloudflare env access:** `import { env } from 'cloudflare:workers'` — only available server-side. Client code accesses bindings through server functions. Types auto-generated by `wrangler types` into `worker-configuration.d.ts`.

## Testing

Two separate test systems:

- **Vitest** (`test/`): Runs in `@cloudflare/vitest-pool-workers` with real workerd runtime. Has own `test/tsconfig.json` extending root config with vitest types. Tests import directly from `cloudflare:workers`.
- **Node native** (`internal-tests/`): Validates hook behavior and project structure using `node --test`. Tests run in Node, not Workers.

## Stop Hook

`.claude/hooks/verify-before-stop.mjs` runs `pnpm lint → typecheck → test:ci` on Stop event. Skips when only docs change or when `stop_hook_active` is set (recursion guard for Codex review gate). Outputs `{ decision: "block", reason }` on failure.

## Conventions

- Path alias: `~/` maps to `src/`
- TypeScript 6 strict mode; `baseUrl` is removed (deprecated in TS6)
- ESLint flat config ignores generated files: `routeTree.gen.ts`, `worker-configuration.d.ts`, `test/env.d.ts`
- `test/` excluded from root tsconfig include — tests use their own tsconfig with `@cloudflare/vitest-pool-workers` types
- `postinstall` auto-runs `wrangler types`; if `worker-configuration.d.ts` errors on install, delete it and rerun `pnpm cf-typegen`
